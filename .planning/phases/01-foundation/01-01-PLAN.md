<task type="auto">
  <name>Task 2: Create public API stubs, types, and error classes</name>
  <files>src/index.ts, src/types.ts, src/errors.ts</files>
  <action>
1. Create src/types.ts with the public type definitions:
   - `ReadOptions` interface: { delimiter?: string; encoding?: string; [key: string]: unknown }
   - `WriteOptions` interface: { pretty?: boolean; delimiter?: string; [key: string]: unknown }
   - Generic type for read results — keep it simple, the runtime determines the actual type
   - Export all types as named exports

2. Create src/errors.ts with three custom error classes:
   - `OmniReadError extends Error` — constructor takes (message: string, public readonly path: string, public readonly format?: string). Set name = 'OmniReadError'. Include an actionable `suggestion` property.
   - `OmniWriteError extends Error` — same pattern with path property
   - `OmniFormatError extends Error` — takes (message: string, public readonly format: string, public readonly suggestion?: string). Used when a format is unsupported or needs a heavier driver.
   - All three set `Error.captureStackTrace` if available (Node-only, guard with typeof check)

3. Create src/index.ts as the main entry point:
   - Import and re-export all types from './types'
   - Import and re-export all error classes from './errors'
   - Export stub async functions with correct signatures:
     - `export async function read<T = unknown>(path: string, options?: ReadOptions): Promise<T>` — throw new OmniReadError('Not implemented', path) for now
     - `export async function write(path: string, data: unknown, options?: WriteOptions): Promise<void>` — throw new OmniWriteError('Not implemented', path) for now
     - `export async function stream(path: string, options?: ReadOptions): Promise<ReadableStream>` — throw new OmniReadError('Not implemented', path) for now
   - No default export (per user decision: named exports only)

4. Run `pnpm build` to verify everything compiles.
5. Run `pnpm check-exports` (publint + attw) to validate the exports map resolves correctly. Fix any issues reported.
  </action>
