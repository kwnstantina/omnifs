  <name>Task 2: Implement path sanitization with traversal rejection</name>
  <files>src/sanitize.ts, src/sanitize.test.ts</files>
  <behavior>
    - sanitizePath('data.json') returns 'data.json' (passthrough for safe paths)
    - sanitizePath('./data/file.csv') returns './data/file.csv' (relative safe)
    - sanitizePath('/absolute/path.json') returns '/absolute/path.json' (absolute safe)
    - sanitizePath('../etc/passwd') throws OmniReadError with message containing "directory traversal"
    - sanitizePath('data/../../etc/passwd') throws OmniReadError with message containing "directory traversal"
    - sanitizePath('data/..\\etc\\passwd') throws OmniReadError (backslash traversal on Windows)
    - sanitizePath('https://example.com/data.csv') returns the URL (URLs pass through — no fs traversal risk)
    - sanitizePath('') throws OmniReadError with message containing "empty"
  </behavior>
  <action>
Create `src/sanitize.ts` exporting `sanitizePath(path: string): string`. The function:
1. Rejects empty strings with OmniReadError("Path cannot be empty", path)
2. If path starts with http:// or https://, return as-is (URLs have no fs traversal risk)
3. Normalize backslashes to forward slashes for cross-platform
4. Check if normalized path contains `..` segment (split by `/`, check for `..` component) — if so, throw OmniReadError with message "Path rejected: directory traversal detected" and the path
5. Return the original path (not normalized — preserve user's intent for the actual fs call)

Write tests first in `src/sanitize.test.ts`, then implement. Import OmniReadError from `../errors.js`.
