  <behavior>
    readBytes (URL string):
    - readBytes('https://example.com/data.json') calls fetch() and returns Uint8Array of response body
    - readBytes with failed fetch (404) throws OmniReadError
    - readBytes with network error throws OmniReadError

    readBytes (Blob/File):
    - readBytes accepts a Blob-like source (detected by checking if input is not a string — handled at facade level, but backend should accept Uint8Array from pre-read Blob)
    - Note: The facade layer will handle Blob→Uint8Array conversion before calling backend. Backend readBytes always receives a path string (URL).

    writeBytes (download trigger):
    - writeBytes creates a Blob from data, creates object URL, creates anchor element, triggers click, revokes URL
    - writeAtomic delegates to writeBytes (no atomic semantics in browser)

    readStream:
    - readStream('https://example.com/large.csv') returns the Response.body ReadableStream
  </behavior>
  <action>
Create `src/backends/browser.ts` exporting `browserBackend: IOBackend`:

**readBytes(path: string):**
1. Call `fetch(path)`
2. If !response.ok, throw `new OmniReadError("Fetch failed: " + response.status + " " + response.statusText, path)`
3. Get arrayBuffer via `response.arrayBuffer()`
4. Return `new Uint8Array(arrayBuffer)`
5. Wrap in try/catch — network errors become OmniReadError

**writeBytes(path: string, data: Uint8Array):**
1. Create `new Blob([data])`
2. Create object URL via `URL.createObjectURL(blob)`
3. Create anchor element: `document.createElement('a')`
4. Set href to object URL, download attribute to `path` (filename)
5. Append to body, click(), remove from body
6. `URL.revokeObjectURL(url)`
Note: path is used as the download filename, not a filesystem path.

**writeAtomic(path: string, data: Uint8Array):**
Delegate to writeBytes — atomic semantics don't apply in browser.

**readStream(path: string):**
1. Call `fetch(path)`
2. If !response.ok, throw OmniReadError
3. If !response.body, throw OmniReadError("Response has no body stream", path)
4. Return response.body (it's already a ReadableStream<Uint8Array>)

For tests: Use vitest with mocked fetch (vi.stubGlobal) and mocked document/window. Test:
- readBytes success: mock fetch returning ok response with arrayBuffer
- readBytes failure: mock fetch returning 404
- writeBytes: mock document.createElement, verify anchor href and click
- readStream: mock fetch returning response with body stream

Note: Do NOT mock at module level. Use vi.stubGlobal('fetch', mockFetch) in each test and restore in afterEach.
